# TEMPLATE - CODING STANDARDS

**Use this template to create or refresh `contexts/coding-standards.md`.**

This template is for authoring. The runtime file should be the active standard Anti-Gravity follows during code work.

---

## WHAT TO CAPTURE

- the coding posture of the project
- naming and file conventions
- implementation defaults
- patterns to follow and avoid
- git and review conventions
- stack-specific conventions once the stack is settled

---

## AUTHORING QUESTIONS

### Coding Posture

- What values should dominate local implementation decisions?
- What review priorities come first in this codebase?

### Conventions

- How are files named?
- How are symbols named?
- What branch and commit style does the team use?
- What code organization defaults should Anti-Gravity follow?

### Guardrails

- Which patterns are preferred?
- Which anti-patterns are explicitly rejected?
- What security and performance floors should always apply?

### Stack-Specific Details

- What conventions only make sense once the actual framework/language choices are known?

---

## RUNTIME FILE TARGET SHAPE

When you write the actual `contexts/coding-standards.md`, keep it in this shape:

1. Runtime Summary
2. Project Coding Posture
3. Naming And File Conventions
4. Implementation Defaults
5. Patterns To Follow
6. Patterns To Avoid
7. Git And Review Conventions
8. Stack-Specific Section
9. Maintenance Notes

---

## FINAL RULE

The runtime `coding-standards.md` should let Anti-Gravity write code that feels native to the project, not bury the important rules under a second philosophy manual.
