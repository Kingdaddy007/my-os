## WHAT THIS WORKFLOW DOES

This workflow ensures performance work starts with measurement, not intuition. It forces the identification of the real bottleneck before any code is changed.

## HIGH-LEVEL PROTOCOL

| Phase | Activity | Goal |
| :--- | :--- | :--- |
| **P1. Define** | Quantify the symptom and target | Establish the baseline and gap |
| **P2. Profile** | Trace the path and find the bottleneck | Identify exactly where time is spent |
| **P3. Options** | Generate targeted improvements | Produce multiple paths for the bottleneck |
| **P4. Decide** | Compare and choose the best path | Select based on ROI and tradeoffs |
| **P5. Execute** | Implement one change at a time | surgically apply the optimization |
| **P6. Verify** | Measure again and check regressions | Prove the gain with numbers |
